Lua Documentation

Introduction

Lua is a portable, lightweight scripting language that was developed back during 1993. Being a small and simple language, lua offers simple, but powerful features. It's used as a component language, for its C API, its numerous libraries (as a result of its dynamic typing). Lua can also act as a glue language that can change the behavior of the components it binds together.

Lua has been used as the leading language in several commercial games (including World of Warcraft), Adobe Lightroom, and many industrial applications. It is also free to use for any purpose. Lua's advantages can be summed up as follows:

  • Lua is fast and portable.
  • Lua is small and embeddable.
  • Lua is both powerful and simple to use.

Variables and Variable Assignment

Variables are names that you assign values to. This is done in order to retrieve or call the value whenever you need. Lua follows the naming conventions for variables that many other programming languages do: Variables can start with underscores and letters, but they cannot have leading numbers. Variables also may not share the same name as special characters or reserved words in the lua language.

Declaring variables in lua is quite simple. You simply type the identifier, followed by a space, and equal sign which assigns the name to its value, and then the value itself. See the example below.

planet = "Saturn"
number = 4
local x,y,z = 3, 2, 1

The last line not only shows how multiple variable assignment is performed, it also uses the keyword "local". This is important when talking about scope, which we will cover in the next section.

Finally, variables can be assigned to any valid type as a value. You can even assign them to expressions. You don't need to assign any value to a variable upon declaring it. When this happens, Lua will treat said variable as nil. 

sum = 400 + 300
print(sum) --returns 700
local n
print(n) --returns nil

Scope

When you declare a global variable in lua, it can be seen and accessed by all of its surrounding blocks, functions, and tables. This behavior can potentially lead to confusion, namespace pollution, and unintended errors. When working with several variables, it's best to declare them with the local keyword in order to lock them to their current chunk or block.

The following example depicts how scope works in lua. Note the values of the variable"n".

n = 4
print(n) --returns 4
do --this starts a new block
    local n = 8
    print(n) --returns 8, value of n local to this block
    n = n * 2
    print(n) --returns 16
end --terminates the block, pushing the variable that was local to it out of scope.
print(n) --returns 4, global n

Functions

Functions are re-usable pieces of code that act as the main method for abstracting expressions and statements. Think of them as machines that perform complex actions with the push of a button, even though we don't see all of the steps the machine takes to get there.

The example below shows a function's syntax, as well as how it's called and what it does.

add = function(num)
    return num + 10;
end
x = 14;
print(add(x)) --returns 24

Functions are called by typing the function name followed by a pair of parenthesis. A function's parameters are listed between the parenthesis if any exist. The end keyword terminates the function and, if no values were returned, the function will return nothing.

It is important to note that functions ar first class values and are subject to the benefits of Lua's many types. They can be table properties, assigned to variables, and can even be passed as parameters to other functions. Functions in Lua can also return multiple values, and with the "..." syntax passed as a parameter, they can take any number of arguments.

Tables

Tables are Lua's one and only data structure. Neither arrays nor objects exist as types in this language, but tables are capable of simulating both. They work kind of like associative arrays where keys can be assigned values, or values can just be indexed.

local tbl = {} --an empty table
local colors = {"red", "pink", "green", "blue"} --a table with indexed values
local heroes = {Superman = "Clark Kent", Batman = "Bruce Wayne", 
                Spiderman = "Peter Parker", Ms_Marvel = "Kamala Khan"}

print(colors[1]) --returns "red"
print(heroes.Ms_Marvel) --returns "Kamala Khan"
print(heroes["Ms_Marvel"]) --returns "Kamala Khan"

local alt = "Ms_Marvel"

print(heroes[alt]) --also returns "Kamala Khan"
print(heroes[Ms_Marvel]) --returns nil

The above example also shows how to access a table's contents. It also shows how the dot notation is simply syntactic sugar for the bracket notation. Note the use of the alt variable in brackets without quotes. When a word is placed between the brackets without quotes, lua will treat it as a variable and "expand" its meaning first.

Flow Control

Lua offers the following constructs for flow control

  • if then...else
  • while
  • repeat
  • Numerical for and Generic for

Example of if then...else: 

if condition then
   --code
elseif condition then
   --code
end

Example of while loop: 

while condition do
   --code
end

Example of repeat: 

repeat --code until condition

Example of for loops. Generic for can be used to iterate over tables.

--Numeric for
for exp1, exp2, exp3 do
   --code
end

--Generic for
for key, value in ipairs(tbl) do
   --code
end

Lexical Scoping and Closures

Lexical Scoping refers to how functions are able to access all local variables and parameters within their enclosing functions. Observe the following behavior.

function big(a,b)
  return function()
    return (a+b)*2
  end
end

medium = big(4,5)
print(medium()) --returns 18

For local variables, enclosed functions will "remember" them even when they've fallen out of scope. This is how a closure operates. It recalls and is able to access the instance of said local variable even when the variable's scope has already passed.

function count()
   local count = 0
   return function()
      count = count + 1
      return count
   end
end

c1 = count()
print(c1()) --returns 1
print(c1()) --returns 2
print(c1()) --returns 3
