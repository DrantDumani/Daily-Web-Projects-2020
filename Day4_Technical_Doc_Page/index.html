Lua Documentation

Introduction

Lua is a portable, lightweight scripting language that was developed back during 1993. Being a small and simple language, lua offers simple, but powerful features. It's used as a component language, for its C API, its numerous libraries (as a result of its dynamic typing). Lua can also act as a glue language that can change the behavior of the components it binds together.

Lua has been used as the leading language in several commercial games (including World of Warcraft), Adobe Lightroom, and many industrial applications. It is also free to use for any purpose. Lua's advantages can be summed up as follows:

  • Lua is fast and portable.
  • Lua is small and embeddable.
  • Lua is both powerful and simple to use.

Hello World

In line with typical programming conventions, we'll run a simple "Hello World" program

print("Hello World");

Variables

Variables are names that you assign values to. This is done in order to retrieve or call the value whenever you need. Lua follows the naming conventions for variables that many other programming languages do: Variables can start with underscores and letters, but they cannot have leading numbers. Variables also may not share the same name as special characters or reserved words in the lua language.

Declaring variables in lua is quite simple. You simply type the identifier, followed by a space, and equal sign which assigns the name to its value, and then the value itself. See the example below.

planet = "Saturn"
number = 4
local x,y,z = 3, 2, 1

The last line not only shows how multiple variable assignment is performed, it also uses the keyword "local". This is important when talking about scope, which we will cover in the next section.

Finally, variables can be assigned to any valid type as a value. You can even assign them to expressions. You don't need to assign any value to a variable upon declaring it. When this happens, Lua will treat said variable as nil. 

sum = 400 + 300
print(sum) --returns 700
local n
print(n) --returns nil

Scope

When you declare a global variable in lua, it can be seen and accessed by all of its surrounding blocks, functions, and tables. This behavior can potentially lead to confusion, namespace pollution, and unintended errors. When working with several variables, it's best to declare them with the local keyword in order to lock them to their current chunk or block.
